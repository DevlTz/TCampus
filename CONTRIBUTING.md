# Contributing

Obrigado por considerar contribuir com o projeto **BPP — Boas Práticas de Programação**!
Este arquivo descreve o fluxo básico para abrir issues e pull requests, e as regras relativas a contribuições.

## Como contribuir
1. Abra uma issue descrevendo o bug, sugestão ou feature. Forneça contexto e passos para reproduzir (quando aplicável).
2. Espere por aprovação do mantenedor ou discussão na issue antes de iniciar grandes mudanças.
3. Crie um branch a partir de `main`: `feature/<descrição>` ou `fix/<descrição>`.
4. Faça commits atômicos com mensagens claras. Inclua testes quando aplicável.
5. Submeta um Pull Request referenciando a issue. Descreva o que mudou, por que mudou e os efeitos colaterais esperados.
6. Todas as PRs serão avaliadas e revisadas; mantenedores podem solicitar alterações.

## Política de licenciamento e uso comercial
- O código está licenciado sob **Business Source License 1.1 (BSL 1.1)**: o código é *source-available* mas possui restrições de uso comercial conforme indicado em `LICENSE`.
- Documentação (guias, PDFs) está licenciada como **CC BY-NC-SA 4.0** (arquivo `LICENSE-DOCS`).

Para uso comercial, redistribuição comercial ou contratos especiais, entre em contato com os mantenedores conforme indicado em `AUTHORS.md`.

## Contributor License Agreement (opcional)
Este repositório recomenda que contribuidores concordem com a política de contribuição: ao enviar um PR, você afirma que tem o direito de contribuir e que sua contribuição pode ser licenciada pelo projeto segundo as licenças acima.

Se você desejar, poderemos adicionar um arquivo de CLA formal (ex.: `CLA.md`) para disponibilizar assinaturas formais.
